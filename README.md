### パズルゲームの基盤とCMS設計とインフラ構築【Unituy/PHP Laravel/AWS】 (2025年1月〜2025年10月)
【プロジェクト概要】モバイル向けパズルゲームの開発プロジェクトにエンジニアとして参画。既存ゲームをベースに、フロントエンド開発とあわせて、APIサーバーおよびCMS（管理機能）を含むバックエンド基盤の設計・実装を担当。具体的には下記。

- フロントエンド開発を担当。UniRxを用いて自作のステートマシンを構築し、ゲーム内のステート遷移を柔軟かつ管理しやすい設計とした。クリア演出時には、ミニストーリー（ADVパート）を再生する機能を実装し、CSVファイルからテキストデータを読み込むことでストーリーが表示される仕組みを構築した。

- APIの設計/開発。
EC2上にLaravel製のAPIサーバを構築し、ユーザー登録・ログイン認証・ユーザー引き継ぎ機能・クエストデータの保存機能を実装。ゲームアセットはS3に保存し、APIを通じてプレイ可能なパズルアセットのみを動的に取得する設計を採用。マスターデータにはバージョン管理機能を実装し、クライアントとのデータ整合性を常に保てる構成とした。

- CMSの設計・開発を担当。マスターデータの更新およびバージョン管理をCMS上から行えるよう設計し、非エンジニアでも新規パズルの設定やパズル画像の追加が可能な仕組みを構築。アセットバンドルを用いずにアセット更新を反映できるようにすることで、運用コストの削減と更新速度の向上を実現。さらに、ユーザーの登録状況やクエストクリア情報もCMS上で確認できるようにし、運営業務の効率化を図った。

- CMS上にボタンを配置し、Googleスプレッドシート上のマスターデータをワンクリックでデータベースに直接インポートできるシステムを構築。これにより、マスターデータ更新の作業を効率化し、業務全体の省力化に貢献した。

- アセットデータの管理を担当。管理用CMSからS3に画像をアップロードした際に、AWS Lambdaを用いて自動的にWebP形式への変換およびサムネイル画像の生成を行う仕組みを設計。WebP形式を採用することで、非エンジニアによるアセット更新作業を容易にし、従来のアセットバンドル化やサーバーデプロイを不要に。これにより、バンドル化を行わずとも軽量な画像をフロントエンド側で効率的にダウンロードできる構成を実現した。

- ローカル開発環境の整備を担当。Dockerを用いて、Laravel API、MySQL、Webサーバーに加え、S3互換ストレージとしてMinIOコンテナを構築。さらに、アップロード時のWebP変換処理を再現するため、Lambda相当のPythonコードを実行できる専用コンテナも作成し、本番環境に近い動作検証が可能なローカル環境を整備した。

- Notion上に、管理CMSによるマスターデータの更新手順や、ローカル開発環境向けの擬似S3構築方法などのマニュアルを作成。業務手順をドキュメント化することで、作業の属人化を防ぎ、チーム全体での運用の再現性と引き継ぎの効率を高めた。
